# Perfect Dismantling

Version: `0.1 Alpha`

Workspace for generating a Witcher 3 Next-Gen 4.04 dismantling-result mod.

This repository intentionally does not include extracted game XML or built bundles. Generate those locally from your own installed copy of The Witcher 3.

## What to edit

The dismantle output table is data-driven in item XML, not in a custom WitcherScript table. The game script calls:

```witcherscript
parts = inv.GetItemRecyclingParts(dismantledItem);
```

Those parts come from item definition blocks like:

```xml
<recycling_parts>
    <parts count="2">Leather straps</parts>
    <parts count="1">Silver ingot</parts>
</recycling_parts>
```

Edit the relevant `gameplay/items/def_item_*.xml` files for normal New Game, and also edit the matching `gameplay/items_plus/def_item_*.xml` files for New Game+.

## Folder Layout

- `uncooked/content0/` contains vanilla XML extracted from the Steam 4.04 install. Treat this as read-only reference.
- `uncooked/dlc_ep1/` contains locally extracted Hearts of Stone reference files.
- `uncooked/dlc_bob/` contains locally extracted Blood and Wine reference files.
- `source/modPerfectDismantling/content/` is the mod source. Generated override XML files live here.
- `dist/modPerfectDismantling/content/` is generated by the build script.
- `scripts/` contains repeatable helper commands.

## Workflow

1. Extract reference XML from your local Next-Gen 4.04 install:

```powershell
.\scripts\Extract-ReferenceXml.ps1
```

2. Generate exact dismantling tables from the extracted Next-Gen 4.04 recipe XML:

```powershell
.\scripts\Generate-PerfectDismantling.ps1
```

3. Find the XML file that owns an item if you want to inspect or manually adjust it:

```powershell
.\scripts\Find-Item.ps1 -Pattern "Bear Boots"
```

4. Optional manual override workflow:

```powershell
.\scripts\New-OverrideFile.ps1 -RelativePath "gameplay\items\def_item_junk.xml"
.\scripts\New-OverrideFile.ps1 -RelativePath "gameplay\items_plus\def_item_junk.xml"
```

5. Edit generated or copied XML in `source/modPerfectDismantling/content/...`.

6. Build an installable mod:

```powershell
.\scripts\Build-Mod.ps1
```

7. Install it into the game:

```powershell
.\scripts\Install-Mod.ps1
```

8. Run Witcher Script Merger after installing. XML-only mods usually avoid script compilation conflicts, but bundled XML conflicts can still happen if another mod overrides the same item XML.

For in-game validation steps, see [TESTING.md](TESTING.md).

## Important Compatibility Notes

- This workspace uses the installed Steam Next-Gen game at `C:\Program Files (x86)\Steam\steamapps\common\The Witcher 3`.
- The local `wcc_lite.exe` is from `WitcherScriptMerger\Tools\wcc_lite\bin\x64`.
- Do not edit `playerWitcher.ws` just to change dismantle output. That would collide with your current merged script stack and is unnecessary for normal dismantle-result changes.
- If another mod changes the same item XML, whichever bundle wins load priority may replace the other. In that case, merge the XML manually by starting from the other mod's changed XML and adding your `<recycling_parts>` changes.

## 0.1 Alpha Limits

- Designed for The Witcher 3 Next-Gen 4.04, not Classic 1.32.
- Generated from official base game, HoS, and B&W XML only.
- Does not automatically support new items added by other mods unless their XML is included in a custom generation workflow.
- Skips ambiguous recipes and stack-output recipes, such as bolts crafted in batches.
